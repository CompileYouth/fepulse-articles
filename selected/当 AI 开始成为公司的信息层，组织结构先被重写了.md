## Jack Dorsey 真正在怀疑的，不是某个岗位，而是“公司”这种结构还能不能继续成立

这期访谈最有意思的地方，不是 Jack Dorsey 又讲了一遍 AI 会很重要，而是他把问题直接推到了组织这一层。他说自己从去年开始，一直同时感到恐惧、希望和兴奋，因为 AI 逼着他重新问一个以前很少会认真问的问题：未来“公司”到底还是不是现在这种样子，层级结构是不是还成立，管理链条是不是还必要。

在他的描述里，过去的层级制并不是天然正确的东西，它本质上只是人类在信息流动能力有限的前提下，发明出来的一种管理方式。公司之所以需要一层层 manager、director、VP，很大程度上不是因为人天生就该被这样组织，而是因为信息没法直接流到所有人那里，只能靠层级在中间转述、过滤和解释。这个前提一旦被 AI 改写，层级本身就不再是不能碰的东西。

## 每条 Slack、每个 PR、每份文档，现在都可以变成公司的“组织记忆”

Jack Dorsey 的核心判断其实非常工程化。因为像 Block 这样的 remote-first 公司，日常工作几乎都会留下 artifact：Slack 消息、邮件、PR、代码、Google Docs、录屏、会议记录。这些以前只是散落在各个工具里的操作痕迹，现在却可以被统一拿来建模。他的意思不是给公司加一个聊天机器人，而是把这些 artifact 变成一个可以被持续查询、持续理解的智能层。

一旦这样做，公司内部就不再只有 CEO 或管理层才能俯瞰全局。任何人理论上都可以直接“和公司对话”，问它现在做得怎样、哪里卡住了、哪些信息正在丢失、哪些方向最有价值。董事会、分析师、管理层甚至一线员工，面对的都不只是零散文档，而是一个能围绕公司本身回答问题的系统。

## “每家公司都是一个 mini-AGI”，真正说的不是模型，而是组织开始被当成可计算系统

Jack Dorsey 提出“每家公司都可以是一个 mini-AGI”，这句话表面上像口号，但他说的不是再造一个通用模型，而是把公司本身当成一个可以建模的智能体。组织过去一直也是一种 intelligence，只不过这种 intelligence 高度依赖人肉传达信息，既低效又高损耗。现在如果所有 artifact 都能沉淀进智能层，公司就第一次有机会用更低损耗的方式运行自己。

他举的例子也很具体。董事会过去每季度要看一堆 board docs 和 slide，但时间依然不够；如果每个董事都能直接去 query 公司的智能层，季度会议就可以少花时间做信息搬运，多花时间做真正重要的判断。分析师、内部团队、不同业务线，也都能用类似方式减少等待、减少层层汇报、减少信息在中途被扭曲的概率。

## Block 想把 6000 人公司的管理层级，压到只剩两三层

这场访谈最值得程序员警惕的一点，是 Jack Dorsey 已经在 Block 内部把这件事往真实组织改造上推了。他说现在从自己到公司任意一个人，中间大约还有五层，他希望今年把这个深度压到两到三层。极限情况下，他甚至说理想形态是所有 6000 人都直接向他“汇报”，只是中间不再靠传统 manager，而是靠那个 intelligence layer 去承接大量信息组织工作。

他同时提到，未来会希望把公司角色尽量归一到少数几种：最核心的是 builder/operator，也就是真正写代码、做产品、做销售、做设计、跑运营的人。这些人会被 agent 强化，一个人能探索的广度和深度都显著提高。人在这里最持久的价值，不再是单纯执行，而是 judgment、taste 和 creativity。

## 对程序员最现实的提醒是：AI 先改写的，也许不是写代码，而是公司怎么流动信息

很多工程师现在谈 AI，仍然先想到 copilot、coding agent 和自动写代码。但 Jack Dorsey 这一套真正锋利的地方，在于它把问题从“程序员怎么提效”推进到了“整个公司如何重新组织信息”。如果这条路成立，未来被压缩的未必只是具体岗位，更可能是那些本质上依赖信息搬运、层层同步和人工转述才能存在的组织层。

对 FEPulse 读者来说，这期最值得记住的不是“每家公司都能变成 mini-AGI”这句标题党式表达，而是它后面那层更硬的判断：**当 AI 可以直接读取公司里的代码、文档、PR、消息和会议记录时，组织结构本身就会变成一层可以被重写的软件。** 到那时，真正重要的人，不再只是会写代码的人，而是能定义 intent、能做判断、能对组织智能层负责的人。
