## 当 Cognition 说出 “12X productivity gains” 时，他们指的不是一点点提效

这期对话里最抓人的一句话，不是“AI 会改变软件行业”，而是 Scott Wu 直接说，当他们在 2024 年推出 Devon 时，**一个人花一小时管理 Devon，往往相当于这个人自己去做 6 到 12 小时的工作量。** 这不是普通意义上的 copilot 提效，而是把人从直接执行里抽出来，转向管理一个持续干活的 agent。

更激进的是，Scott Wu 说在 Cognition 内部，工程师现在已经“不再亲手写代码”了。这里当然不是字面意义上任何代码都不碰，而是说角色重心已经明显迁移：从自己一行行实现，变成把任务拆给 agent、检查结果、重新定义目标、决定下一步怎么推进。换句话说，真正值钱的环节正在从“亲手敲”转向“如何把智能系统调到最有效”。

## 软件 abundance 到来后，工程师、设计师、产品经理先感受到的不是协作，而是边界被冲掉

这期访谈里还有一句很刺激的话：当软件进入 abundance 时代，工程师、设计师和产品经理会互相看着对方说，“我不再需要你了”。这句话不是说这些角色会真的消失，而是说它们原来靠职责分工维持的边界，正在被 agent 大幅侵蚀。

如果一个人可以靠 agent 快速生成原型、补设计稿、搭功能、写代码、改文案、连通工作流，那么很多过去必须在多人之间来回传递的信息，现在都能被压缩在一个人和一组 agent 之间完成。岗位不会立即消失，但岗位之间的接缝会先被冲烂，原来靠“这是设计的活”“这是工程的活”来划分责任的方式，开始越来越站不住。

## 对年轻工程师更有利的，不一定是经验，而是没有旧工作流包袱

Scott Wu 对一个问题的回答很值得程序员认真想一下：AI coding tool 时代，年轻人是不是反而更有优势？他的答案基本是肯定的。因为在软件工程这件事上，所有人对 agent 的经验都很浅，工具每三个月就要换一次代际，你之前积累的很多固定工作流，反而可能变成包袱。

他提到，内部很多最能把 agent 用出效果的人，本来就是十几岁、刚从高中出来，或者几乎没有既有工程工作流预设的人。原因不神秘，他们没有太多“事情本来就该这样做”的想法，所以更容易完全拥抱一套新工作方式。这和很多公司担心“初级工程师会被 AI 吃掉”形成了一个反差：被先淘汰的未必是经验少的人，反而可能是最依赖旧范式的人。

## AI 公司里最硬的要求，开始变成“每个人都得像 chief engineer 一样理解全栈”

Russell 这段回忆也很关键。他说自己在 Tesla Autopilot 做机器学习时，Elon 一直强调一句话：**everyone is chief engineer**。放到 AI 公司里，这句话反而更重要，因为随着模型、训练、推理、产品和工具链越来越纠缠，原本那些清晰的抽象边界正在变得越来越脆弱。

这意味着，未来真正强的工程师未必只是某一个窄点上的高手，而是能理解模型能力、工具行为、产品目标和系统限制如何连在一起的人。AI 不是让技术深度不重要了，而是逼着更多人把深度从单点扩展成跨层理解。你不一定要自己从头实现所有东西，但你必须知道整条链路为什么会成、为什么会崩。

## 对 FEPulse 读者最现实的提醒是：Agent 先重排的不是代码量，而是你在团队里的价值定义

这场对话真正让人不舒服的地方，不是 Devon 到底是不是已经足够强，而是它逼着大家重新面对一个问题：如果软件产能继续被 agent 抬高，一个人在团队里的价值到底还剩什么？Scott Wu 给出的答案其实已经非常明确了。最重要的不是你能不能亲手完成更多执行，而是你能不能定义目标、管理 agent、做判断、跨角色推进，并且在技术快速变化时不断重建自己的工作流。

对程序员来说，这意味着下一轮竞争可能不是“谁写代码更快”，而是“谁更早把自己从单点执行者升级成能调度智能系统的人”。AI Agent 把软件产能抬高之后，最先消失的不是程序员，而是原来那些看起来很牢固的岗位边界。
