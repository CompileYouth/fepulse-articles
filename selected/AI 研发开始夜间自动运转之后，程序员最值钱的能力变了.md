## Eric Schmidt 真正想提醒的，不是奇点口号，而是程序员已经被迫换工作方式了

这场对话表面上聊的是机器人竞赛、能源短缺和奇点时间表，但最值得程序员记住的，其实是 Eric Schmidt 对当下软件研发方式已经变化到什么程度的描述。他说旧问题已经不是“AI 会不会写代码”，而是**今年就是 agent 的年份**，整个湾区都在围绕 reasoning 和 agent 的爆发重新组织研发流程。真正令人震动的，不是哪个模型分数更高，而是大家已经默认让系统自己跑起来，然后第二天再去看结果。

他给的判断很直接：现在最重要的变化不是上下文窗口，而是模型的 reasoning skill 和“能持续思考多久”。这意味着 agent 不再只是帮你补几行代码，而是可以在更长时间里持续展开任务、反复尝试、按评价函数优化结果。也正因为这样，软件研发开始从“人亲手一步步做”，转向“人定义方向，系统持续执行”。

## 从“我来写代码”变成“我写 spec 和 evaluation function”，这是更本质的迁移

Eric Schmidt 讲了一个特别具体的例子。他认识的一位年轻程序员做 UI 时，不再是坐在那里一行行改，而是先把自己想要的东西写成 spec，再写一个 test function，也就是 evaluation function，晚上 7 点把系统打开，然后去吃饭、睡觉，第二天早上起来再看系统在夜里跑出了什么结果。

这段话最关键的不是“AI 帮他提效很多”，而是研发主语已经变了。以前程序员的核心动作是亲手实现，现在更像是在定义目标、定义约束、定义评估方式，然后让 agent 在足够多硬件支持下自己搜索方案。Eric 说这种事以前要他和 Google 里十个程序员做六个月，现在一个人睡一觉就能看到雏形。真正被放大的，是人的 intent 和 evaluation，而不是单纯的打字速度。

## 这会让顶级程序员更值钱，而不是更不值钱

很多人会把这件事理解成“程序员完了”，但 Eric Schmidt 的判断正好相反。他说最顶级的程序员会变得更值钱，因为这些系统现在仍然需要人来控制，而最能控制它们的人，恰恰是那些真正理解并行化、系统边界、评估方式和抽象层的人。过去顶尖程序员就比下一层的人贵很多，未来这个差距可能还会继续拉大。

原因也不难理解。agent 越强，越不意味着所有人自动变强，反而更考验谁能把任务写清楚、把评估写对、把系统拉到正确轨道上。你不再只是 coder，而更像一个“程序系统的导演”。真正决定产出的，不是你手有没有更快，而是你能不能把一个开放问题转成机器可持续优化的问题。

## 组织结构也会跟着裂开：未来会是少数超大公司，加大量极小团队

Eric Schmidt 还给出了一个对行业结构很关键的判断：未来很可能会同时出现两种极端组织。一边是少数非常大的公司，因为只有它们能长期吃下硬件、电力和资本密集型投入；另一边是大量非常小的公司，因为在 agent 时代，做很多产品已经不再需要过去那么多人。

这对 FEPulse 读者的现实意义是，AI 不只是改变“你怎么写代码”，还在改变“你需要多少人才能开一家公司”。一边是基础设施门槛越来越高，另一边是小团队用 agent 驱动产品的门槛越来越低。程序员将越来越多地站在两个极端之间：要么进入超级平台，要么进入极小但极高密度的团队。

## 对程序员最现实的提醒是：以后最重要的不是你会不会写，而是你会不会定义

这场访谈里最值得反复咀嚼的一句，不是奇点会不会在两三年内到来，而是 Eric Schmidt 自己那种近乎震惊的感受：他看着这些系统，发现已经“没有什么是我能做而它做不了的”。这并不意味着程序员消失，而是意味着程序员的价值中心开始转移。未来真正值钱的，不再只是能亲手把某一块写出来的人，而是能定义 spec、定义 evaluation、理解 agent 如何并行工作并把它们约束到正确方向上的人。

对 FEPulse 读者来说，这期最重要的结论可以压成一句话：**当 AI 研发开始在你睡觉时继续自动运转，程序员最值钱的能力就从“执行”转向“定义问题和控制系统”。** 谁更早完成这次迁移，谁就会在下一轮生产力分化里占到便宜。
